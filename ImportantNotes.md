# Important notes to never forget
## Rules - Read at least 1nce per day.
1. Never forget to stage commits before pushing them.
2. Unless you are confident (ie familiar with Terminal) use GUI for Git-CMDS
    - Recommendation: GitCola (easiest to use)
3. This branch is a work in progress and will eventually use GitFlow layout.
    - Different fixes/updates/etc goes under diff branches to avoid confusion.
    - Always follow standard commit messaging layout to explain commits (essential)
4. If not familiar use Grep often and available Git commands for debugging.
    - Grep: Helps search code faster (command line unfortunately)
    - Git : Helps establish new ways to debug code easier than standard GUI.
5. Faster tests are personalized tests over GUI Debugger (overtime this wastes
   valuable time.) Making tests is actually faster in Longrun.
    - Important we establish a formal Testing procedure to test code as we push
6. Always collaborate and never assume any question is stupid.
    - Everything we are doing is either hard or confusing therefore we are all
equally dumb so dont sweat it.
